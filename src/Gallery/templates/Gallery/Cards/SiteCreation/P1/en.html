Building This Website

Quick Overview

I built this site with Django. The main feature is a simple blog, and it also supports localization. The idea was both to practice my skills and to have a place to share some info about myself.

The database runs on PostgreSQL. I handled everything on my own: design, development, and deployment. Requests go through nginx, served by gunicorn. The site runs on HTTPS with a free certificate from Let’s Encrypt. The whole project is packed into Docker Compose and starts with a single command. It’s hosted on my personal server, with my own domain.

I also set up a small CI/CD pipeline: whenever I push to main, the server automatically updates files and restarts the containers.

Story

This is my first Django project that actually made it to a real release. I had tried Django before, but my projects always stayed unfinished. I experimented with an async server, websockets, a small chat, and even a tiny game. But this blog is the one I finally brought online.

Why “part 1”? Well, I do plan to keep working on it. It’s both fun and potentially useful. But plans don’t always turn into reality.

I actually wrote the site about half a year ago, but only deployed it recently — first I had exams, then I went home to relax, and other stuff kept getting in the way.

A Bit About the Process

Development itself wasn’t too hard — Django turned out to be a solid and well-designed framework. Deployment though, that’s where the challenges came in.

First, I needed a server. I didn’t want to use my main PC, so I decided to go with a Raspberry Pi. That came with its own issues: I needed a case (so I 3D-printed one), and later found out I’d placed it in a bad spot — right next to my desktop PC and a radiator, so it was overheating. I had to add a fan. Big thanks to my dad for helping with electronics advice.

With the server ready, I hit another problem: my router Xiaomi Mi Router 4C blocks port 80, and you can’t disable it. The only solution (short of buying a new router) was flashing OpenWRT (link). Took me a while to build up the courage, but in the end it was pretty simple. I needed port 80 for Let’s Encrypt and HTTPS to work properly.

After that, it still took me about a week to finish setting things up in my free time: wrapping everything in Docker, configuring nginx, and polishing the environment. And now here we are — you’re reading this post.
